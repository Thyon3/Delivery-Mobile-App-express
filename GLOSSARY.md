# Glossary

## Technical Terms

### API (Application Programming Interface)
Interface that allows different software applications to communicate with each other.

### REST (Representational State Transfer)
Architectural style for building web services using HTTP methods.

### JWT (JSON Web Token)
Compact, URL-safe means of representing claims to be transferred between two parties.

### ORM (Object-Relational Mapping)
Technique for converting data between incompatible type systems (database and programming language).

### PostGIS
Spatial database extender for PostgreSQL that adds support for geographic objects.

### WebSocket
Communication protocol providing full-duplex communication channels over a single TCP connection.

### Rate Limiting
Technique to control the rate of traffic sent or received by a network interface.

### Middleware
Software that acts as a bridge between an operating system or database and applications.

### CORS (Cross-Origin Resource Sharing)
Mechanism that allows restricted resources on a web page to be requested from another domain.

### Redis
In-memory data structure store used as a database, cache, and message broker.

## Application-Specific Terms

### Customer
End user who orders food through the platform.

### Driver
Delivery personnel who picks up and delivers orders.

### Restaurant Owner
Business owner who manages restaurant, menu, and orders.

### Order State Machine
System that manages order status transitions through defined states.

### Geolocation Query
Database query that finds locations within a specified distance using coordinates.

### Spatial Index
Database index optimized for location-based queries (GIST index).

### Optimistic Locking
Concurrency control method that assumes multiple transactions can complete without affecting each other.

### Background Job
Task that runs asynchronously without blocking the main application.

### Webhook
HTTP callback triggered by specific events in external services.

### Hot Reload
Development feature that automatically restarts the server when code changes.

## Database Terms

### Migration
Version-controlled changes to database schema.

### Schema
Structure that defines how data is organized in a database.

### Transaction
Sequence of database operations that are treated as a single unit.

### Index
Database structure that improves the speed of data retrieval operations.

### Foreign Key
Field that creates a link between two tables.

## Performance Terms

### Caching
Storing frequently accessed data in fast-access storage.

### N+1 Query Problem
Performance issue where an application makes one query followed by N additional queries.

### Connection Pooling
Maintaining a pool of database connections that can be reused.

### Query Optimization
Process of improving database query performance.

### Load Balancing
Distributing workload across multiple servers.

## Security Terms

### Hashing
One-way transformation of data into a fixed-size string.

### Argon2
Modern password-hashing function.

### SQL Injection
Code injection technique used to attack data-driven applications.

### XSS (Cross-Site Scripting)
Security vulnerability that allows attackers to inject malicious scripts.

### CSRF (Cross-Site Request Forgery)
Attack that forces users to execute unwanted actions on a web application.
